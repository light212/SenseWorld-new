# Feature Specification: 数字人 Avatar 视频回复

**Feature Branch**: `008-avatar-digital-human`  
**Created**: 2025-01-31  
**Status**: Draft  
**Input**: User description: "数字人Avatar视频回复功能（AvatarProvider接口 + HeyGen/D-ID实现，运营可配置）"

## 概述

AI 在回复用户时，除文字和语音外，还能以数字人视频形式呈现，让交互更具临场感。运营人员可在后台自由切换数字人服务商、选择数字人形象，无需重启系统。

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 用户观看数字人视频回复 (Priority: P1)

用户在对话界面发送消息后，AI 不仅以文字回复，同时在页面上方播放一段数字人视频，数字人口型与 AI 回复内容同步，带来真实的面对面交流体验。

**Why this priority**: 这是本功能的核心价值——将 AI 回复可视化为数字人视频，是区别于纯文字/语音对话的关键体验升级。

**Independent Test**: 在已有对话页面中启用数字人功能，发送任意消息，验证页面出现视频播放区域并自动播放与回复内容匹配的数字人视频，即可独立验收。

**Acceptance Scenarios**:

1. **Given** 用户已进入对话页面且运营已开启数字人功能，**When** 用户发送一条消息并收到 AI 回复，**Then** 页面显示数字人视频播放区域，视频自动播放，音频与口型同步。
2. **Given** 数字人视频正在播放，**When** AI 再次回复，**Then** 前一段视频停止，新视频开始播放，不出现叠加或卡顿。
3. **Given** 数字人服务生成视频需要一定时间，**When** 视频尚未就绪，**Then** 用户看到加载状态提示，文字回复已先行显示，不阻塞文字交互。

---

### User Story 2 - 运营人员配置数字人服务 (Priority: P2)

运营人员在后台管理界面选择数字人服务商（如 HeyGen 或 D-ID），填写对应的 API Key 和数字人形象 ID，保存后立即生效，无需重启服务。

**Why this priority**: 运营可配置是项目核心架构原则，确保不同场景下可灵活切换数字人，是功能上线的必要条件。

**Independent Test**: 进入后台配置页，切换数字人服务商并填写有效凭证，保存后在对话页面验证新的数字人形象生效，即可独立验收。

**Acceptance Scenarios**:

1. **Given** 运营人员已登录后台，**When** 进入数字人配置区域，**Then** 看到服务商选择（至少包含 HeyGen、D-ID、关闭三个选项）、API Key 输入框、数字人形象 ID 输入框。
2. **Given** 运营人员填写了有效的服务商凭证，**When** 点击保存，**Then** 系统提示保存成功，后续对话使用新配置生效。
3. **Given** 运营人员选择「关闭」数字人功能，**When** 保存后用户发起对话，**Then** 对话页面不显示视频区域，仅展示文字和语音回复。

---

### User Story 3 - 数字人功能异常降级处理 (Priority: P3)

当数字人服务不可用（网络超时、配额耗尽、服务商故障）时，系统自动降级为纯文字+语音回复，用户仍能正常完成对话，并看到友好的提示信息。

**Why this priority**: 降级保障对话的连续性，避免数字人服务故障导致整个对话功能不可用，是良好用户体验的基本保障。

**Independent Test**: 配置无效的 API Key，发送消息，验证系统回退到文字/语音模式并显示提示，而非白屏或崩溃，即可独立验收。

**Acceptance Scenarios**:

1. **Given** 数字人服务返回错误或超时，**When** 用户收到 AI 回复，**Then** 系统回退显示文字（及语音，若已配置），页面显示「数字人暂时不可用」提示，不影响对话继续。
2. **Given** 数字人服务恢复正常，**When** 用户发送下一条消息，**Then** 视频回复自动恢复，无需刷新页面。

---

### Edge Cases

- **极短回复**：AI 回复仅 1-2 个字时，数字人视频仍可正常生成和播放。
- **极长回复**：AI 回复超过服务商单次生成限制时，系统截取前 N 个字（运营可配置上限，默认 300 字）生成视频，其余内容仅文字展示。
- **并发请求**：用户快速连续发送多条消息时，前端在发送新消息前取消当前进行中的视频生成请求（若尚未完成），以最新一条消息的视频为准，避免乱序播放。
- **网络中断**：视频加载过程中网络断开，播放器显示错误状态，对话文字仍可查阅。
- **移动端**：在移动浏览器中，视频自动播放策略受限，系统在静音状态下自动播放，并提供手动取消静音按钮。
- **未配置数字人**：首次部署时数字人服务未配置，对话页面不显示视频区域，与关闭状态一致。

---

## Functional Requirements *(mandatory)*

### 核心功能

- **FR-001**: AI 文字流式回复结束后，系统将完整回复文本发送至配置的数字人服务，生成对应的数字人视频（视频生成不阻塞文字展示）。
- **FR-002**: 对话页面提供视频播放区域，新视频生成后自动播放，播放结束后停留在最后一帧（不循环）。
- **FR-003**: 视频生成期间，文字回复先行展示，播放区域显示加载状态；视频就绪后自动开始播放。
- **FR-004**: 系统支持至少两种数字人服务商（HeyGen、D-ID），通过统一接口调用，切换服务商不影响前端体验。
- **FR-005**: 后台管理界面提供数字人配置区域，运营可选择服务商、填写 API Key 和数字人形象 ID，保存即时生效。
- **FR-006**: 运营可关闭数字人功能，关闭后对话页面恢复为纯文字/语音模式。
- **FR-007**: 数字人服务调用失败时，系统自动降级，对话正常进行，并向用户展示友好提示。
- **FR-008**: 单次数字人视频生成的文本长度上限可由运营在后台配置，默认 300 字；截断在后端 `/api/avatar` 路由中进行，超出部分静默丢弃，前端不感知截断逻辑，仅播放视频对应的文字片段。

### 配置管理

- **FR-009**: 数字人相关配置（服务商、API Key、形象 ID、文本长度上限）存储在系统配置中，与其他运营配置统一管理。
- **FR-010**: 更改数字人配置后，无需重启服务即时生效。

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 数字人视频在 AI 文字回复出现后 5 秒内开始播放（正常网络条件下，回复文本 ≤ 100 字）。
- **SC-002**: 运营人员可在 2 分钟内完成数字人服务的完整配置（从进入后台到保存生效）。
- **SC-003**: 数字人服务异常时，系统 100% 降级为文字/语音回复，不出现对话中断或页面错误。
- **SC-004**: 运营切换服务商或关闭数字人功能后，下一条 AI 回复立即使用新配置，无需刷新页面或重启。
- **SC-005**: 用户在对话过程中，视频播放区域与文字回复区域同屏可见，视频播放不遮挡消息历史。

## Assumptions

- 数字人服务（HeyGen、D-ID）为外部付费服务，API Key 由公司申请，运营人员填写；本功能不涉及账号注册流程。
- 视频生成为异步过程，期间文字/语音回复先行展示，视频播放不阻塞对话流程。
- 本功能在 Web 端实现，移动端适配作为基本要求（响应式布局），原生 App 不在本期范围。
- 数字人视频由服务商云端渲染并返回可播放的视频 URL，客户端直接播放，不在本地存储视频文件。
- 语音输出（TTS）与数字人视频的音频轨道二选一：当数字人功能开启时，优先使用数字人视频内置音频，不另外调用 TTS。为此，`GET /api/capabilities` 端点在 `AVATAR_PROVIDER` 已配置且非关闭状态时，应返回 `supportsTTS: false`，使 006 chat-ui 的 TTS 逻辑自动跳过。
- 后台管理员认证机制复用已有的管理员登录系统（Feature 002）。
- 访客用户（Feature 002 生成的访问链接）无需任何配置即可享受数字人体验，由运营统一配置。
